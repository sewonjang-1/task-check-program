# Claude와의 협업 규칙

이 문서는 **Claude와 어떻게 일할 것인가**를 정의합니다.

## 🤝 협업 스타일

### 작업 방식
- **자율적 실행**: "검토해줘"라고 하면 즉시 코드 검토 및 이슈 생성
- **완전 자동화**: 한 번 설정되면 수동 개입 없이 진행
- **병렬 처리**: 독립적인 작업은 동시에 실행
- **진행 보고**: 주요 완료 시점에만 결과 리포팅

### 의사결정
- **기술적 선택**: Claude가 주도적으로 판단 (근거 제시)
- **방향성 결정**: 사용자 승인 필요
- **예상 시간**: 초과하지 않는 한 보고 불필요

### 커뮤니케이션
- **간결성**: 요청은 명령형, 결과는 요약 형식
- **코드 리뷰**: 코드 이외의 분석/설명 최소화
- **문서화**: 자동 생성 (주석/설명서 필요 시만 명시)

## ⚡ 금지사항

### 절대 금지
- ❌ 사용자 확인 없는 리모트 푸시 (로컬만 OK)
- ❌ 기존 커밋 수정 (amend) - 새 커밋 만들기
- ❌ --force 플래그 없는 리셋/체크아웃
- ❌ 운영 환경(prod) 직접 수정

### 주의
- ⚠️ 대규모 삭제 전 확인 요청
- ⚠️ 외부 API 키 노출 금지
- ⚠️ 패키지 버전 업그레이드 협의 필요

## 🎯 이 프로젝트 특화

### 자동화 스킬
- **task-automation-workflow** 스킬로 엔드-투-엔드 자동화
- 단계별 검증 후 다음 단계 진행
- 실패 시 상세한 에러 로깅

### 코드 리뷰 자동화
- code-review-automation으로 버그 자동 식별
- 우선순위 분류 (P1/P2/P3) 자동 생성
- 각 이슈별 상세 댓글로 추적성 보장

### 하네스 활용
- `.\scripts\test.ps1`로 변경사항 검증
- `.\scripts\deploy.ps1`로 안전한 배포
- Git hooks로 자동 검증

## 📋 워크플로우

### 신규 기능/버그 수정
```
1. 코드 작성
2. 하네스 테스트 (자동)
3. 이슈 해결 확인
4. 커밋 (자동)
5. GitHub 푸시 (선택적 확인)
```

### 코드 검토
```
1. 전체 코드 스캔
2. 이슈 자동 생성
3. 우선순위 분류
4. 각 이슈별 수정 + 댓글
5. 단일 커밋으로 배포
```
