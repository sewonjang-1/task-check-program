# 업무 자동화 완전 워크플로우

이 스킬을 사용하면 **팀 전체의 업무 관리를 자동화**하고, 
**높은 코드 품질을 유지**하면서 **빠르게 배포**할 수 있습니다.

## 🎯 3단계 워크플로우

### Phase 1: 시스템 구축 🏗️

**목표**: 팀의 업무 관리를 위한 완전한 시스템 구축

**포함되는 것**:
- ✅ 웹 기반 업무 관리 UI
  - 대분류 → 소분류 → 상세내용 3단계 계층 구조
  - 일일/주간/월간 업무 자동 분류
  - 월별 진척도 시각화
  
- ✅ 자동 이메일 보고
  - 매일 오후 5시 미완료 업무 발송
  - HTML 템플릿 기반 정규 보고서
  - Windows Task Scheduler 자동 등록
  
- ✅ 반복 업무 자동화
  - 일일/주간/월간 반복 설정
  - 자동으로 업무 생성
  - 중복 방지 로직
  
- ✅ 데이터 관리
  - JSON 기반 로컬 저장 (서버 불필요)
  - 데이터 내보내기/가져오기
  - 브라우저 저장소 자동 관리

**소요 시간**: 30분 (설정 포함)

**결과**: 
```
manage_ui.html          # 웹 UI
tasks.json             # 데이터
email_config.json      # 이메일 설정
incomplete_scheduler.* # 자동 실행 스크립트
logs/                  # 실행 로그
```

---

### Phase 2: 코드 검토 🔍

**목표**: 구축된 시스템의 버그 식별 및 품질 개선

**검토 항목**:

1️⃣ **보안 검토**
- XSS 취약점 확인 (innerHTML 남용)
- 입력값 검증 부족
- 민감한 정보 노출

2️⃣ **성능 검토**
- 대량 데이터 처리 시 렉
- 불필요한 재렌더링
- 날짜 계산 최적화

3️⃣ **에러 처리 검토**
- try-catch 누락
- 파일 I/O 에러 미처리
- 네트워크 장애 대응

4️⃣ **데이터 무결성 검토**
- null/undefined 체크
- 타입 검증
- 경계값 처리
- 날짜 형식 검증

5️⃣ **코드 품질 검토**
- 중복 코드
- 함수 길이
- 네이밍 컨벤션
- 문서화 수준

**출력**:
```
이슈 #1: 🐛 [P1] 월말 날짜 계산 버그
이슈 #2: ⚠️  [P2] JSON 파싱 에러 미처리
...
이슈 #9: ⚡ [P3] 성능 최적화
```

**소요 시간**: 1시간 (9개 이슈 발견)

---

### Phase 3: 버그 수정 & 배포 ✅

**목표**: 발견된 이슈를 수정하고 GitHub에 배포

**프로세스**:

```
각 이슈별로:
  1. 코드 수정 (버그 해결)
  2. 커밋 생성 (Fixes: #N)
  3. 이슈 댓글 (수정 완료 기록)
  4. GitHub 푸시 (배포)
```

**예시**:
```javascript
// #3 JSON 파싱 에러 수정
function loadTasks() {
    try {
        const tasks = localStorage.getItem('tasks');
        return tasks ? JSON.parse(tasks) : [];
    } catch (error) {
        showNotification('데이터 로드 실패', 'error');
        return [];
    }
}
```

**결과**:
```
✅ 7개 P1/P2 이슈 수정 완료
📝 각 이슈에 상세 댓글 기록
🔗 GitHub에 변경사항 푸시
📊 수정 통계: +150줄, 3개 파일 변경
```

**소요 시간**: 2시간 (7개 이슈 수정)

## 🚀 사용 방법

### 단계 1: 워크플로우 시작

```
업무 자동화 시스템을 구축하고 
자동으로 품질 검토까지 해줘
```

### 단계 2: Phase 1 완료 후

```
자동으로 생성된 manage_ui.html을 브라우저에서 열어서
업무를 추가해보세요
```

### 단계 3: 자동 진행

워크플로우가 다음을 자동으로 수행합니다:
1. ✅ Phase 2: 코드 검토 및 이슈 생성
2. ✅ Phase 3: 버그 수정 및 배포

### 단계 4: 검증

```
gh issue list                    # 이슈 확인
git log --oneline -10          # 커밋 확인
git push origin main           # 수동 푸시 (이미 자동 완료)
```
