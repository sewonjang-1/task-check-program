# 프로젝트 철학 & 원칙

이 문서는 **이 프로젝트가 왜 존재하고 무엇을 지향하는가**를 정의합니다.

## 🎯 핵심 목표

**팀이 업무에만 집중하도록, 관리의 모든 것을 자동화한다.**

- 업무 입력 → 자동 분류 → 자동 보고 → 자동 추적
- 사람이 할 일: 업무 추가 + 완료 체크
- 시스템이 할 일: 모든 나머지

---

## 💎 설계 원칙

### 1. **자동화 우선**
```
수동 = 누락 가능성
자동 = 빠뜨릴 수 없음
```
하나의 자동화로 1000명이 혜택을 본다.

### 2. **단순함**
```
3단계: 입력 → 분류 → 보고
복잡함: 설정 > 10개 이상
```
설정이 많으면 안 쓴다.

### 3. **신뢰성**
```
자동화 = 신뢰 기반
신뢰 = 에러 처리
```
버그하나 = 신뢰도 전부 잃음

### 4. **투명성**
```
모든 결정이 로그에 기록됨
모든 변경이 커밋에 남음
모든 문제가 이슈로 추적됨
```

---

## 🏗️ 아키텍처 철학

### 서버리스 설계
```
관리 부담 최소화
Windows + LocalStorage로 충분
```
팀 규모 < 100명이면 추가 인프라 불필요.

### 로컬 우선
```
tasks.json (JSON)
email_config.json (설정)
logs/ (기록)
```
모두 프로젝트 디렉토리에 완결.

### 확장 가능
```
현재: localStorage (단일 팀)
미래: DB 전환 가능 (다중 팀)
```
하나의 하네스로 양쪽 지원.

## 📊 성공의 정의

### 단기 (1개월)
- ✅ 팀 전체가 사용 중
- ✅ 매일 자동 이메일 발송
- ✅ 0건의 수동 개입

### 중기 (3개월)
- ✅ 미완료 업무 0건 유지 (자동 알림 덕분)
- ✅ 월별 진척도 50% 이상 달성
- ✅ 새로운 팀도 복제해서 사용 가능

### 장기 (1년)
- ✅ 회사 전체 표준 시스템
- ✅ 1000명 이상 팀이 사용
- ✅ 자동화 덕분에 업무 효율 30% 증가

## 🔄 진화 방향

### Phase 1 (완료) ✅
- 웹 UI + 자동 이메일 + 진척도
- 3개 스킬 생성
- 완전 자동 하네스 구축

### Phase 2 (미래)
- **대시보드**: 실시간 팀 현황 표시
- **분석**: 팀의 업무 패턴 분석
- **예측**: AI로 완료 예상 날짜 예측
- **협업**: 팀 간 의존성 자동 추적

### Phase 3 (비전)
- **통합**: 캘린더/슬랙/이메일과 자동 연동
- **지능화**: 우선순위 자동 추천
- **최적화**: 팀 역량에 맞춘 일정 조정

## 🛡️ 핵심 가치

### 신뢰
완전 자동화만이 신뢰를 만든다.
수동 체크는 결국 하게 되고, 하지 못하면 불신이 생긴다.

### 투명성
모든 기록이 남아야 한다.
"뭐 했어?"라는 질문이 없어야 한다.

### 단순함
팀이 이해할 수 없으면 쓰지 않는다.
가장 복잡한 것은 설정이 아니라 "사용하지 않음".

### 공정성
모든 팀원을 동등하게 대한다.
자동화는 편애 없다.
